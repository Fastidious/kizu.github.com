Custom Specificity

In previous article about conditions for CSS variables I talked about things that could be used rather soon, even if the support is not there yet. In this article I'll go even further — I'll be talking about one thing from the CSS Selectors Level 4, which is even less adopted yet, but we still can experiment on things that're already there in some browsers.

The part of this spec I'll be looking at today is the new, enhanced :not(). It would be really cool.

The All-New Negation Pseudoclass

In Selectors Level 3 :not() could have only a single simple selector inside of it. The leveled-up version allows for so much more!

The specification for the :not() doesn't say much about its new features, almost all it says is that instead of a simple selector you can put a selector list inside. But that alone means a lot.
